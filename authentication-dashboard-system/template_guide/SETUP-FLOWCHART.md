## Common Stopping Points

If you need to pause setup, good stopping points are:

1. **After Initial Setup** (Step 1)
   - All config files updated
   - Dependencies installed
   - Can commit to git

2. **After Cloudflare Resources** (Step 3)
   - D1 database created
   - KV namespaces created
   - IDs saved in wrangler.toml

3. **After Database Init** (Step 4)
   - Schema applied
   - Admin user created
   - Can test database queries

4. **After Secrets** (Step 6)
   - All secrets configured
   - Ready to deploy
   - Can test locally

5. **After Deployment** (Step 8)
   - Worker and Pages deployed
   - Working on default URLs
   - Can test full stack

## Troubleshooting Quick Guide

### Setup Issues
- **Can't find wrangler command**: Install globally with `npm install -g wrangler`
- **Permission denied**: Run `wrangler login` first
- **Database not found**: Check database_id in wrangler.toml matches

### Deployment Issues
- **Build fails**: Check for TypeScript errors, run `npm run build` locally
- **Worker deploy fails**: Verify account_id and all secrets are set
- **CORS errors**: Check CORS_ORIGIN secret matches frontend URL exactly

### Runtime Issues
- **Emails not sending**: Verify BREVO_API_KEY, check Brevo account status
- **Login fails**: Check database has admin user, password is correct
- **Rate limiting issues**: Verify KV namespace IDs in wrangler.toml
